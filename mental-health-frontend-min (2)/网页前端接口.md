# 心理健康系统 - 后端接口 API 文档（前端需求说明）

本文档由前端项目逆向整理，描述前端对后端接口的依赖、请求方式、**传入参数（含类型与必填说明）**、**返回参数（含类型与说明）**及期望的响应格式，供后端开发实现与联调使用。

---

## 一、通用约定

### 1.1 基础地址与鉴权

- **基础地址**：`http://localhost:8080`
- **请求头**：
  - `Content-Type: application/json`
  - 需要登录的接口：`Authorization: Bearer <token>`
- **Token 来源**：登录成功后后端返回 token，前端存本地（localStorage），后续请求时写入 Authorization。

### 1.2 统一响应格式

所有接口返回格式统一为：

```json
{
  "code": 200,           // 200 成功，401 未授权，其它为业务或系统错误
  "data": { … },         // 业务数据
  "message": "操作成功"  // 提示文案
}
```

**错误码规范**

- 200：成功
- 400：参数错误
- 401：未授权/登录失效
- 403：无权限
- 404：资源不存在
- 500：服务器内部错误
- 1001~1999：自定义业务错误

在失败情况下 data 可以为空，message 用于前端提示。

## 二、认证与用户

### 2.1 管理端登录

- **路径**：`POST /api/auth/admin/login`
- **描述**：管理员/心理中心等后台用户登录

**传入参数（Body）**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 账号 |
| password | string | 是 | 密码 |
| role | string | 是 | 角色（admin/center/tutor/counselor/...） |

**返回参数（data）**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | string | 用户 ID |
| username | string | 账号 |
| name | string | 姓名 |
| role | string | 角色 |
| token | string | 鉴权令牌 |

**成功响应示例**

```json
{
  "code": 200,
  "data": {
    "id": "admin_1",
    "username": "admin",
    "name": "管理员",
    "role": "admin",
    "token": "mock-token-123"
  },
  "message": "登录成功"
}
```

**失败示例**

```json
{
  "code": 401,
  "msg": "账号或密码错误"
}
```

## 七、数据结构

### 7.1 预约状态枚举

```typescript
export type AppointmentStatus =
  | 'draft'            // 草稿（刚创建）
  | 'info_done'        // 来访登记完成
  | 'scale_done'       // 前测量表完成
  | 'sign_done'        // 知情同意书已签署
  | 'completed'        // 整个预约完成
  | 'form_completed'   // 已完成来访登记 & 前测
  | 'consent_signed'   // 已签署知情同意书
  | 'submitted'        // 已提交
  | 'confirmed'        // 已确认
  | 'cancelled'        // 已取消
  | 'checked_in'      // 已签到
  | 'report_done'    // 咨询报告完成
  | 'closed'         // 结案
```

### 7.2 预约数据结构

```typescript
export interface Appointment {
  id: string                 // 预约id
  studentId: string          // 学生id
  counselorId: string        // 咨询师id
  counselorName: string      // 咨询师名字
  appointmentDate: string    // 预约日期
  appointmentTime: string    // 时间段

  status: AppointmentStatus  // 当前状态

  create_time: string        // 创建时间
  update_time: string        // 最近更新时间

  visitInfo?: {
    reason: string
    emergencyContact: string
  }

  scaleResult?: {
    mood: string
    stressLevel: number
  }

  timeline: {
    status: AppointmentStatus
    time: string
  }[]

  signAt?: string
}
```
