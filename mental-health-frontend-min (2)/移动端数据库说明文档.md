## 一、文档说明与使用指南

### 1.1 文档目的

本文档基于前端项目（Mock 数据、数据模型、API 文档）逆向整理，汇总**前端所需的数据结构与字段**，供后端设计与实现数据库时参考。

### 1.2 使用方式

| 步骤 | 说明 |
|------|------|
| ① 字段对照 | 表格中列出前端接口使用的字段名、类型、必填及说明，后端请尽量保持一致或提供映射 |
| ② 比对反馈 | 若后端已有数据库设计，请将差异填写在 [第六章](#六与后端比对) 表格中，并反馈前端 |
| ③ 枚举统一 | 所有枚举值（status、role、level 等）已在 [第五章](#五枚举值速查) 统一列出，请保持一致 |

### 完整表清单

| 模块 | 表名 | 说明 |
|------|------|------|
| 用户 | `users` | 系统用户（学生/家长/教师/辅导员/咨询师/领导/管理员） |
| 组织 | `colleges` `majors` `grades` `classes` | 学院→专业→年级→班级 四级结构 |
| 咨询 | `students` `teachers` `consultants` | 学生、辅导员、心理咨询师 |
| 咨询 | `schedules` `schedule_periods` | 咨询师排班与时段 |
| 咨询 | `appointments` `appointment_records` | 预约记录、咨询过程记录 |
| 测评 | `assessments` `assessment_questions` `assessment_answers` `assessment_results` | 量表、题目、作答、结果 |
| 活动 | `activities` `activity_registrations` | 团体活动及报名 |
| 自助 | `self_help_articles` `article_likes` `article_collections` | 心理文章及点赞/收藏 |
| 危机 | `crisis_cases` `crisis_interventions` `crisis_level_history` | 危机个案及干预记录 |
| 微课 | `micro_courses` `course_chapters` `course_progress` | 课程、章节、学习进度 |
| 消息 | `notices` `parent_messages` `messages` | 通知、家长留言、站内信 |
| 其他 | `monthly_reports` `feedback` `conversation_records` `parent_student_bindings` | 月报、反馈、谈心谈话、家长子女绑定 |

## 四、表间关系

```
users ←→ students / teachers / consultants  (user_id 或 account)
colleges → majors → grades → classes → students
teachers ←→ classes  (managed_class_ids 或 teacher_classes)
consultants → schedules → schedule_periods
appointments → students, consultants, schedule_periods
appointments → appointment_records  (1:1)
crisis_cases → crisis_interventions, crisis_level_history
crisis_cases ← assessments, appointments  (通过 student_id)
```

## 五、枚举值速查

### 5.1 用户角色 role

| 值 | 说明 |
|----|------|
| student | 学生 |
| parent | 家长 |
| teacher | 辅导员/教师 |
| counselor | 心理咨询师 |
| part_time_counselor | 兼职心理咨询师 |
| college_leader | 院系领导 |
| school_leader | 校领导 |
| center_admin | 中心管理员 |
| system_admin | 系统管理员 |

### 5.2 预约状态 status

| 值 | 说明 |
|----|------|
| pending | 待确认 |
| confirmed | 已确认 |
| completed | 已完成 |
| cancelled | 已取消 |
| no_show | 已爽约 |

### 5.3 排班时段状态 status

| 值 | 说明 |
|----|------|
| available | 可预约 |
| booked | 已预约 |
| leave | 请假 |
| unavailable | 不可用 |

> 前端 API 中可能使用 free/booked/full，可与后端约定映射。

### 5.4 危机/风险等级 level

| 值 | 说明 |
|----|------|
| red | 红色 · 极高危 |
| orange | 橙色 · 高危 |
| yellow | 黄色 · 中危 |
| blue | 蓝色 · 轻度 |
| green | 绿色 · 正常 |

### 5.5 活动状态 status

| 值 | 说明 |
|----|------|
| upcoming | 即将开始 |
| ongoing | 进行中 |
| finished | 已结束 |

### 5.6 测评题目类型 type

| 值 | 说明 |
|----|------|
| scale | 量表题（1-5 等级） |
| single | 单选题 |
| multi | 多选题 |

### 5.7 站内消息类型 type

| 值 | 说明 |
|----|------|
| appointment | 预约相关 |
| assessment | 测评相关 |
| crisis | 危机相关 |
| system | 系统通知 |
| activity | 活动相关 |
